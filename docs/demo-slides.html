Why it matters

Office lunch is operational work disguised as chat.

  • 01Organizers chase meal choices across threads, DMs, and late replies.
  • 02Budgets and dietary rules are easy to state but hard to enforce manually.
  • 03Approval and placement need a clean handoff before the final order is sent.
  • 04The best place to coordinate is where the office already responds: Slack.

How the live demo works

Slack reaches the local backend through ngrok.

Step 1

Slash command

Slack sends a signed request to the public ngrok command URL.

Step 2

Node backend

The app verifies Slack signatures and creates the lunch run.

Step 3

Block Kit

Participants choose meals through Slack buttons and menus.

Step 4

Organizer gates

Only the organizer can lock, approve, and place the order.

Step 5

Mock placement

The demo returns a provider order id without using private Swiggy APIs.
